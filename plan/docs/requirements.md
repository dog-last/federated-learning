# 需求分析

## 功能需求

### 必做功能

#### 1. 基础功能

| 功能模块 | 描述 |
|----------|------|
| 模型与数据加载 | 使用轻量CNN模型，加载分割好的CIFAR-10数据集 |
| TCP通信 | 基于Python原生socket库建立TCP连接，解决粘包问题 |
| 联邦学习逻辑 | Server广播模型 → Client本地训练 → Server聚合更新 |
| 网络状态监测 | 实时输出轮次、耗时、Payload大小等网络日志 |

#### 2. 扩展功能1：去中心化通信（10分）

- 采用**环形拓扑**进行P2P通信
- 实现**故障跳过机制**：节点故障时自动跳过，保持环形传播
- 完全不依赖中心Server

#### 3. 扩展功能2：网络延迟与掉队者处理（10分）

- 模拟弱网环境：`time.sleep(5)` 人为延迟
- Server端超时机制：超过阈值则放弃该节点
- 使用已收到节点进行聚合，保障系统进度

### 选做功能（预留接口）

| 功能 | 描述 | 分值 |
|------|------|------|
| Split Federated Learning | 模型分割，Client-Server协同计算 | 10-15分 |
| 跨设备真实部署 | 局域网内多PC真实部署 | 10分 |

## 技术栈要求

### 核心技术

| 类别 | 技术 | 用途 | 备注 |
|------|------|------|------|
| 网络通信 | `socket` | TCP连接建立与数据传输 | **禁止使用HTTP框架** |
| 序列化 | `pickle`, `io.BytesIO` | 模型参数序列化/反序列化 | - |
| 深度学习 | `torch`, `torchvision` | 模型定义、训练、CIFAR-10 | - |
| 配置管理 | `pyyaml` | YAML配置文件解析 | - |
| 日志 | `logging` | 结构化日志输出 | - |

### 代码质量

| 工具 | 用途 |
|------|------|
| `ruff` | Linting + 格式化 |
| `pre-commit` | Git Hooks 自动检查 |

### 禁止事项

- ❌ 禁止使用HTTP框架（Flask, FastAPI等）
- ❌ 禁止使用高层RPC库（gRPC, Thrift等）
- ❌ 禁止通过本地磁盘文件共享传递模型
