# 自定义开放问题 Proposal：SplitFed 与通信压缩扩展

## 1. 问题背景

联邦学习的核心价值在于：多个数据拥有者无需上传原始数据，也可以协同训练同一个模型。但普通联邦学习仍存在一个现实矛盾：数据通常分布在边缘设备或个人终端上，而这些数据拥有者未必具备足够的本地算力。FedAvg 往往要求每个 Client 保存并训练完整神经网络，再上传完整模型参数或参数更新。对于手机、嵌入式设备或低功耗终端来说，这会带来较高的计算压力、内存占用和通信开销。

因此，联邦学习虽然缓解了“数据不能集中”的问题，但并没有完全解决“数据拥有者训练能力不足”的问题。我们的开放问题正是围绕这个矛盾展开：如何让弱算力 Client 也能参与隐私协同训练？

## 2. 开放问题内容

我们计划采用文献中称为 **SplitFed** 的方法，将神经网络从某个中间层切开：

- Client 只保存并运行前半部分网络。
- Server 保存并运行后半部分网络。
- Client 不上传原始样本，而是上传切分层产生的中间激活值。
- Server 基于中间激活值继续前向传播、计算损失并反向传播。
- Server 将切分层处的梯度返回给 Client。
- Client 使用该梯度继续反向传播，更新本地前半部分模型。

这样，一次完整的训练仍然可以被端到端地完成，但完整模型不再全部运行在 Client 上。

## 3. SplitFed 的可行性

SplitFed 的可行性主要来自三点。

1. 神经网络天然具有层级结构，可以从中间层进行较干净的切分。切分点前后的两段计算图不需要共享原始输入，只需要通过激活值和对应梯度连接。也就是说，前半段模型与后半段模型可以部署在不同机器上，通过 TCP 消息协作完成前向传播和反向传播。

2. 中间激活值通常不像原始图片、文本或标签那样直接可读。传输激活值在一定程度上保护了能起到联邦学习的重要目标之一——保护隐私。（不过这一点并不等价于严格隐私保证；如果后续需要更严谨的隐私分析，可以进一步引入差分隐私、噪声注入等加强手段。）

3. Client 只负责前半段模型计算，可以减少本地计算量和模型存储压力。Server 承担后半段模型训练、损失计算和训练协调，更符合“弱 Client + 强 Server”的边缘计算场景。

## 4. SplitFed 系统设计

在我们的系统中，原始 CNN 被拆分为两部分：

- `SplitClientCNN`：部署在 Client，负责前置卷积层和初级特征提取。
- `SplitServerCNN`：部署在 Server，负责后续卷积层、全连接层和分类输出。

每轮训练流程如下：

1. Server 向各 Client 广播当前轮次和模型状态。
2. Client 对本地 batch 执行前半段前向传播，得到 smashed data，即切分层激活值。
3. Client 通过 TCP 发送 `split_batch` 消息，包含中间激活值、标签和必要的轮次信息。
4. Server 接收激活值后，完成后半段前向传播、loss 计算和反向传播。
5. Server 通过 `split_grad` 消息返回切分层处的梯度。
6. Client 根据收到的梯度继续反向传播，更新本地前半段模型。
7. 一轮结束后，Client 上传前半段模型更新，Server 聚合得到下一轮客户端侧全局模型。

## 5. 通信压缩

传输速度目前仍然是 SplitFed 系统的潜在瓶颈。我们将进行 gzip 压缩尝试，但由于神经网络权重是高熵、接近随机的，我们预估 gzip 的压缩效果很可能不佳。若后续继续优化，更合理的方向可能是量化或随机量化，例如把 float32 激活值/梯度映射为 int8/uint8，并在需要隐私增强时结合差分隐私思想加入受控随机噪声。由于工作量，量化部分不一定能有充足的时间实现。

## 6. 工作量与分值申请

我们认为该自定义开放问题的工作量和创新性足以申请开放题满分 **25 分**。

1. SplitFed 需要重新设计模型结构、训练流程和通信消息类型，把原本完整运行在 Client 上的 CNN 拆分为 Client-side model 和 Server-side model。

2. SplitFed 引入更复杂的通信模式：Client 发送激活值和标签，Server 返回梯度。相比普通 FedAvg 每轮只上传一次模型更新，这要求系统在训练过程中持续处理更细粒度的消息交互。

3. Server 端需要承担后半段模型训练、loss 计算、梯度回传、客户端侧模型聚合和网络日志记录。Client 端也需要支持前半段模型训练、跨机器反向传播、本地评估和轮次级模型上传。两端逻辑都需要围绕 SplitFed 重新组织。

综上，我们提出的开放任务要求在原有联邦学习系统上增加一个完整的训练模式。该扩展同时体现了网络协议、分布式训练、隐私、弱客户端场景和系统监控分析，因此我们申请开放题 **25 分**。
