# 测试与后续开发建议

## 1. 现有测试

项目当前主要有两类测试：

- `tests/test_pipeline.py`：覆盖数据准备、模型初始化、归一化和配置加载。
- `tests/test_tcp_smoke.py`：覆盖 TCP 报文帧格式的冒烟测试。

## 2. 建议的验证方式

```bash
python -m unittest discover -s tests
```

如果只想快速检查核心文件能否导入和编译，也可以先运行项目已有的语法检查或冒烟测试。

## 3. 新增功能时的检查顺序

1. 先确认配置是否需要改动。
2. 再确认数据格式是否仍然兼容。
3. 然后检查服务端、客户端和通信协议是否同步更新。
4. 最后补充测试，避免只改了运行路径却没有验证。

## 4. 适合继续扩展的方向

- 新的数据集支持。
- 更多客户端或更复杂的异构网络设置。
- 更丰富的监控指标与更细的训练阶段可视化。
- 更严格的通信协议校验与失败恢复逻辑。
