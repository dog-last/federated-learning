# 《基于 TCP 通信的联邦学习系统》展示主旨与讲稿

## 一、展示主旨

我们的项目围绕“如何用纯 TCP socket 支撑联邦学习中的大模型参数传输”展开。基础目标是实现 1 个中心 Server 与 3 个 Client 的联邦训练闭环：Server 广播全局 CNN 模型，Client 只读取自己的本地数据并训练，随后将模型权重通过 TCP 字节流回传，Server 收齐后进行加权聚合，并在每轮输出网络传输日志和测试准确率。

在基础功能之上，我们还实现了若干扩展：去中心化 Ring 拓扑、SplitFed 分割联邦、弱网延迟与掉队者处理、实时监控面板，以及 gzip 压缩开关。最终三种模式在 MNIST 测试集上均达到约 99% 准确率，超过作业要求的 85%。

展示时建议强调三点：

1. 这不是用 HTTP/RPC 封装出来的模拟系统，核心消息都通过 Python 原生 `socket` 传输。
2. TCP 粘包问题由“8 字节长度头 + 4 字节魔数 + pickle payload”的自定义协议解决。
3. 弱网不是只写了 `sleep`，而是从配置进入 Client，影响更新返回时机，Server 通过超时和 `min_clients` 决定是否跳过慢节点继续聚合。

## 二、3 分钟展示讲稿

各位老师、同学好，我们展示的是“基于 TCP 通信的联邦学习系统”。

这个题目的核心不是单纯训练一个 MNIST 分类模型，而是在网络层面把联邦学习真正跑起来。我们的系统里有一个中心 Server 和三个 Client。每个 Client 只能读取自己的 `client_x_data.pt`，不能接触全局数据或其他客户端数据。Server 只负责下发模型、收集更新、加权聚合和评估全局模型。

第一部分是通信协议。由于神经网络参数序列化后通常有几百 KB，直接用 `recv(1024)` 会遇到 TCP 字节流没有消息边界的问题。我们在 `core/communicator.py` 里统一实现了封包格式：先发送 8 字节大端整数表示 payload 长度，再发送 4 字节魔数 `SF26`，最后发送 pickle 序列化后的消息体。接收端先精确读 8 字节拿到长度，再精确读 4 字节校验魔数，最后用循环一直收，直到收满指定长度。这样可以避免模型参数截断、多个消息粘在一起或者反序列化失败。

第二部分是基础联邦学习流程。Client 启动后先向 Server 注册。Server 等三个 Client 都在线后进入训练轮次。每一轮 Server 把全局 CNN 的 `state_dict` 广播给所有 Client；Client 加载这个权重，在自己的本地数据上训练 1 个 epoch，并计算本地训练、验证、测试指标；然后把训练后的 `state_dict`、样本数和指标发回 Server。Server 收到更新后按样本数做 FedAvg 加权平均，覆盖全局模型，再在服务端测试集上评估。如果准确率达到目标值，就广播 shutdown 结束训练。

第三部分是弱网和掉队者处理。配置文件中 `network.stragglers.client_2.delay = 5.0`，所以 `client_2` 每轮本地训练完成后，发送更新前会主动延迟 5 秒，并向监控系统上报 `straggler_delay`。Server 并不会无限等待，它在 `_wait_round_updates` 中设置 deadline，最多等待 `server_timeout` 秒。超时后，Server 会统计已经收到哪些 Client 的更新，没收到的标记为 `straggler_dropped`。如果收到的数量不少于 `min_clients`，这一轮仍继续聚合；如果低于 `min_clients`，就中止训练。也就是说，这里模拟的是真实网络中慢节点拖慢全局同步的问题，并给出了可配置的容错策略。

第四部分是扩展。Ring 模式废弃中心 Server，三个节点组成 `Client1 -> Client2 -> Client3 -> Client1` 的环。节点收到前驱模型后先加载权重，再用自己的数据训练，然后通过 TCP 传给后继。Node 1 作为发起节点，在模型绕环一周回来后进行全局测试。SplitFed 模式则把 CNN 切成客户端前半段和服务端后半段，客户端传激活值，服务端计算 loss 和梯度，再把梯度传回客户端继续反向传播，因此通信量明显更高。

最后看实验结果。Centralized 模式第 6 轮达到 0.9900 测试准确率，通信量约 58 MB，耗时约 192.5 秒；Ring 模式第 9 轮达到 0.9901，通信量约 43.5 MB；SplitFed 第 4 轮达到 0.9904，但通信量约 9 GB。结论是：集中式方案最符合本题基础要求，效率也最好；Ring 证明了去中心化拓扑可行；SplitFed 收敛轮次少但网络负担很重，体现了网络架构选择对训练效率的直接影响。

## 三、Poster 展示时的 60 秒版本

我们的系统用纯 Python socket 实现联邦学习通信。基础模式是 1 个 Server 加 3 个 Client，Server 广播全局 CNN，Client 只用本地数据训练，再把模型权重回传，Server 按样本数加权聚合。

为了解决 TCP 粘包，我们设计了统一封包格式：8 字节 payload 长度、4 字节魔数 `SF26`、以及 pickle 序列化后的模型数据。接收端用循环精确读满每一段，因此模型不会因为 `recv` 分片而截断。

弱网方面，Client 可按配置延迟或丢弃某轮更新；Server 通过超时等待和 `min_clients` 判断是否跳过掉队者继续聚合。监控系统会记录每轮耗时、payload 大小、收发消息数和掉队事件。

扩展方面，我们实现了 Ring 去中心化拓扑和 SplitFed 分割联邦。实验结果显示三种模式都达到约 99% 准确率，其中 Centralized 总耗时最短，SplitFed 准确率略高但通信量最大，Ring 证明了无中心节点训练也能跑通。
